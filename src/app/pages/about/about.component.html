The following Interface got implemented in the application:

Variables with the "?" postfix are optional and types like "Status" or "ContactPoint" are custom sub-types defined in the model. The main purpose of the application is to understand the basics of Web frameworks, Angular, and Firebase.

The app only uses built in form validators and firebase error handling, it does not have any other front, or backend validation. The address and information you type in gets saved without checking because of this in case of invalid input Google may return invalid, or no destination at all.
